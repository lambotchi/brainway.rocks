Blog for Free Using Your Own Domain

This guide is for those who have their own domain. If you don’t have one follow [Start a Free Blog Using yourblog.github.io]

THIS IS A DRAFT

Prerequisites

  • Github Account
  • Git 2.8
  • Install Hugo
  • Your own domain. ( You can buy one at Namecheap )

Point your DNS server to Github Pages

  1. Go to your DNS server page (e.g. Namecheap)

  2. Add the following records:

    • A record for @ pointing to 185.199.108.153
    • A record for @ pointing to 185.199.109.153
    • A record for @ pointing to 185.199.110.153
    • A record for @ pointing to 185.199.111.153

    This usually takes up to 30 minutes to take efffect

When a someone hit yourdomain.com or anything.yourdomain.com or yourdomain.com/something they will be pointed to one of the 4 github pages servers

Setup github repository

  1. Create github repository. You can name it whatever you want or as simple as blog or yourdomain.com
  2. Go to the repository’s Settings > Github Pages section > Custom domain
  3. Add your domain (e.g. yourdomain.com)

Github repository will serve like a public directory of your blogs content.

Initialize new hugo site

terminal/command prompt:

> hugo new site yourdomain.com-blog
> cd yourdomain.com-blog
> git init

Choose your theme

Choose from tons of hundred of hugo themes. Take note of your chosen theme’s github URL and name, we will use it for next step.

Hugo is a static site generators, it is not limited to blog style of websites.

Apply your chosen theme

terminal/command prompt:

> git submodule add $CHOSEN_THEME_GITHUB_URL themes/$CHOSEN_THEME_NAME
> echo 'theme = "$CHOSEN_THEME_NAME"' >> config.toml

Sample values:

  • $CHOSEN_THEME_GITHUB_URL: https://github.com/budparr/gohugo-theme-ananke.git
  • $CHOSEN_THEME_NAME: ananke

Add contents

terminal/command prompt:

> hugo new posts/my-first-post.md

Change the title and add some more content below posts/my-first-post.md

Test your blog locally

> hugo server -D

By default this will be served at http://localhost:1313 Ctrl + C to stop. Ideally, you should keep the server running and edit your blog. You will see live results as you save your contents

-D without giving this flag, you only see the contents with draft: false

Show it to the world

Let say that you have finalize your blog and ready to publish it online

  1. Update draft value to false in posts/my-first-post.md

  2. Delete public folder if existing

  3. Add the URL of the repository you created from Setup github repository

    git add submodule https://github.com/lambotchi/todelete.git public

  4. Render public contents. hugo or hugo -D if you want to generate site for your post that are still in draft

  5. Add CNAME record. echo yourdomain.com >> CNAME

  6. Git commit. git add . && git commit -m 'Create first post'

  7. Publish. cd public && git push -u origin master
